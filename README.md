# College Units Fantasy — Backend Setup

Full Next.js + Supabase app with auth, league creation, and invite flow.

---

## Quick Start (30 min to running locally)

### 1. Install dependencies
```bash
npm install
```

### 2. Create Supabase project
1. Go to [supabase.com](https://supabase.com) → New Project
2. Pick a name, region, password → Create
3. Wait ~2 minutes for it to spin up

### 3. Run the database schema
1. In Supabase Dashboard → **SQL Editor** → **New Query**
2. Paste the contents of `supabase/migrations/001_initial_schema.sql`
3. Click **Run** — you should see "Success"

### 4. Configure auth
In Supabase Dashboard → **Authentication** → **URL Configuration**:
- **Site URL**: `http://localhost:3000`
- **Redirect URLs**: Add `http://localhost:3000/auth/callback`

For magic links to work in dev, also go to:
**Authentication** → **Email Templates** → make sure "Confirm signup" is enabled.

### 5. Set up environment variables
```bash
cp .env.local.example .env.local
```
Then fill in your values from Supabase Dashboard → **Settings** → **API**:
```
NEXT_PUBLIC_SUPABASE_URL=https://xxxx.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=eyJ...
SUPABASE_SERVICE_ROLE_KEY=eyJ...   # Settings → API → service_role
NEXT_PUBLIC_APP_URL=http://localhost:3000
```

### 6. Run it
```bash
npm run dev
```
Open [http://localhost:3000](http://localhost:3000)

## How the App Works

### Auth flow
- User clicks "Create League" or "Sign In"
- **Magic Link** tab: enters email → gets a login link → clicks it → auto-signed in
- **Password** tab: traditional email + password
- **Google**: OAuth via Supabase (enable in Dashboard → Authentication → Providers → Google)
- After auth, redirected back to wherever they came from

### League creation
1. User fills 4-step wizard: name → size → buy-in → draft type
2. On submit: row inserted into `leagues` table, invite code auto-generated by DB trigger
3. Commissioner added as first `league_member` with `draft_slot: 1`
4. Share page shows invite link + code + share buttons

### Joining a league
- User visits `/join/[CODE]` (via link or entering code on home page)
- Sees league preview card: name, size, buy-in, spots remaining
- If not logged in: prompted to authenticate first
- Enters team name → clicks Join → added to `league_members`
- Redirected to league dashboard

### League dashboard (`/league/[id]`)
- Shows all members in draft order
- Empty slots shown for unfilled spots
- Realtime updates: when someone joins, their row appears instantly (Supabase Realtime)
- Commissioner sees "Start Draft" button
- Invite section stays visible while `status = 'forming'`

## Database Schema Overview

| Table | Purpose |
|---|---|
| `profiles` | User display names, auto-created on signup |
| `leagues` | League settings, invite code, status |
| `league_members` | Who's in each league + their roster |
| `draft_picks` | Every pick made during the draft |
| `weekly_scores` | Calculated weekly scores per team |
| `matchups` | Head-to-head matchup pairings + scores |
| `playoff_games` | Wild card / semi / final bracket games |
| `invite_uses` | Tracks who used which invite code |

All tables have **Row Level Security (RLS)** — users can only see data for leagues they're in.

## File Structure

```
app/
  page.tsx                  ← Landing + dashboard (logged in/out)
  layout.tsx                ← Root layout + fonts
  auth/callback/route.ts    ← Handles magic link / OAuth redirects
  join/[code]/page.tsx      ← Invite link page
  league/[id]/page.tsx      ← League dashboard
  api/leagues/route.ts      ← POST/GET leagues
  api/join/route.ts         ← POST join a league

components/
  auth/AuthModal.tsx         ← Magic link + password + Google sign-in
  league/CreateLeagueWizard.tsx  ← 4-step league creation flow

lib/
  supabase-browser.ts       ← Supabase client (use client components)
  supabase-server.ts        ← Supabase client (server components / API routes)
  design.ts                 ← Color tokens + global CSS

types/index.ts              ← All TypeScript interfaces

supabase/migrations/
  001_initial_schema.sql    ← Full DB schema with RLS policies
```
